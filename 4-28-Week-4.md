### Brief summary

Arrakis is an experimental research operating system that aims to reduce overhead by removing the kernel form I/O interactions.

### List the major contributions (what is it that's novel) compared to previous work

Previously many server applications would "traverse OS kernel multiple times per client request", Arrakis solves this by providing user-level I/O by creating an architecture that removes the kernel from the I/O path. Arrakis also has two versions of itself, Arrakis-P, which features a POSIX compatible interface, and Arrakis-N, which features a native interface that supports true zero-copy I/O, a feature not normally present in previous operating systems. Arrakis also expands on work that U-Net implemented twenty years ago by using "SR-IOV, the IOMMU, and supporting adapters to provide direct application-level access to I/O devices." They also added 33,786 lines of code to the Barrelfish Operating System code base to implement Arrakis, providing support for SR-IOV, physical and virtual function drivers for the Intel 82599, Extended Message Signaled Interrupts (MSI-X), the Intel IOMMU, the Intel RS3 family of RAID controllers, and several POSIX APIs (for threads, sockets, and epoll). On top of that they created their own user-level netowrk stack, Extaris, and storage API layer, Caladan.

### Strengths / Weaknesses of their approach

Arrakis has many strengths when it comes to I/O, as it removes kernel overhead from the process. As stated in the introduction, this proves to have 2x better read latency, 5x better write latency, and 9x better write throughput compared to Linux.

A weakness I noticed of the operating system was the fact that it needed specific hardware in order to function to its fullest extent. The rich filtering semantics that allow the operating system to divide network traffic into VNICS are not present in all NICs. They specifically had to nerf their own operating system to use MAC addresses as filter arguments as the NICs present on their device, the Intel 82599, can only filter based on source or destination MAC adresses and VLAN tags. They state that if they had more modern or high-performing NICs such as the Solarflare 10Gb adapters, they would be able to filter packets on all header fields.
