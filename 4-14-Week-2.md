# Week 2
### What I Did
This week I really wanted to get going on a feature that I could start researching and working on for Twizzler. I talked a bit with Akshay and Rishit about what they were doing to get a idea of what areas were being worked on and it seems like we're all interested in different parts of the project. I specifically remember Akshay being interested in `zero-copy` and generally getting network features on Twizzler since he has some extra knowledge on that. The unimplemented feature that caught my eye was development tools, specifically, a command line text editor. It seemed simple enough to port over an existing command line text editor written in rust, and I'm also very curious as to how the `libc`/`mlibc`/`abi` process works.

With a feature in mind, I also began writing my Request For Comment (RFC) for the feature and researching. I looked through the various submodules attached to the Twizzler project. I went into more a more thorough search through [Twizzler's patched `mlibc` branch](https://github.com/twizzler-operating-system/mlibc/tree/twizzler) to see how normal C syscalls are re-routed into Twizzler syscalls. The command line text editor I want to attempt to implement, [`femto`](https://github.com/frapa/femto), is rust-based, only 300 lines of code and has only one dependency [`termion`](https://github.com/redox-os/termion), which relies on `libc`, but only has
Four functions that it calls:
* `ioctl`
* `tcgetattr`
* `tcsetattr`
* `isatty`

One struct:
* `termios`

Three constants:
* `STDOUT_FILENO`
* `TIOCGWINSZ`
* `TCSANOW`

One type:
* `c_ushort`

I have forked my own version of the main Twizzler repository and have been working on my own branch to develop support for `femto`. I have also forked Twizzler's patched `mlibc` repository to add support for any missing functions, variables, structs, or types I come across.

### Troubles

So far, femto "runs" when used in the Twizzler terminal, but panics immediately after running into an error with `tcgetattr` stating "Unsupported terminal". I suspect this has to do something with the Twizzler version of the function not being implemented fully or something going wrong in mlibc.
